## 🎯 快速开始

### 启动 OpenClaw

**所有平台通用：**
```bash
# 启动 OpenClaw 服务
openclaw start

# 等待服务启动
# 看到以下信息即表示成功：
# ✅ OpenClaw Gateway started on port 8080
# ✅ Web UI available at http://localhost:8080
```

### 访问 Web UI

**浏览器访问：**
```
http://localhost:8080
```

**首次访问体验：**
1. 浏览器自动跳转到登录页（如已配置）
2. 或直接进入主界面
3. 看到欢迎欢迎界面和示例任务

### 🎯 最佳实践建议

#### 1️⃣ 明确任务描述

**✅ 推荐：**
```
分析这周的销售数据，重点查看客单价和订单数，
生成对比图表并发送邮件给我。
```

**❌ 避免：**
```
看下周的数据。
```

#### 2️⃣ 使用结构化指令

**✅ 推荐：**
```
任务：分析销售数据
数据源：sales_db
时间段：2026-04-01 至 2026-04-06
输出格式：PDF + 邮件
接收人：you@example.com
```

**❌ 避免：**
```
看下数据，发我。
```

#### 3️⃣ 分步执行复杂任务

**✅ 推荐：**
```
第一步：收集数据
第二步：分析趋势
第三步：生成报告
第四步：发送邮件
```

**❌ 避免：**
```
把所有事情都做一遍。
```

#### 4️⃣ 使用预设模板

**✅ 推荐：**
```
使用「周报模板」生成周报，包含销售额、订单数、
同比增长，邮件发给我。
```

**❌ 避免：**
```
生成一个周报，随便写写。
```

## 🐛 常见问题

### Q1: 如何查看历史对话？

**回答：**
```bash
# 使用侧边栏查看会话列表
# 或命令行：
/sessions list
/sessions switch 02
```

### Q2: 如何重新开始对话？

**回答：**
```bash
# 清空当前对话
/clear

# 或创建新会话
/new session "新会话名称"
```

### Q3: 如何停止正在执行的任务？

**回答：**
```bash
# 停止当前任务
/stop
```

### Q4: 如何查看执行日志？

**回答：**
```bash
# 查看最近 20 条日志
/logs -n 20

# 查看特定任务日志
/logs task:analyze_sales
```

### Q5: 界面不响应怎么办？

**回答：**
```bash
# 检查服务状态
openclaw status

# 查看系统日志
openclaw logs

# 如果服务无响应，重启服务
openclaw restart
```
