## 🔧 常见错误

### 1. GPG 签名失败

**现象：**
```
错误：gpg 无法为数据签名
[GNUPG:] PINENTRY_LAUNCHED 51255 curses 1.3.2
gpg: 签名时失败：Inappropriate ioctl for device
```

**原因：** 
- 非 TTY 环境下无法交互
- GPG 配置冲突

**解决方案：**

**方案 A - 禁用自动签名（推荐用于 CI/CD）：**
```bash
git config --global commit.gpgsign false
```

**方案 B - 生成非交互式密钥：**
```bash
# 创建密钥参数文件
cat > gpg-key-params <<EOF
Key-Type: RSA
Key-Length: 4096
Name-Real: 你的名字
Name-Email: your@email.com
Expire-Date: 0
%no-protection
%commit
EOF

# 生成密钥
gpg --batch --gen-key gpg-key-params

# 配置 Git 使用
git config --global user.signingkey YOUR_KEY_ID
git config --global commit.gpgsign true
```

**验证：**
```bash
git commit -m "测试提交" --allow-empty
git log -1 --show-signature
```

### 2. GitHub 认证失败

**现象：**
```
fatal: could not read Username for 'https://github.com'
Password authentication is not supported for Git operations
```

**原因：**
- GitHub 已禁用密码认证
- Token 过期或权限不足

**解决方案：**

**方案 A - 使用 Personal Access Token：**
```bash
# 1. 生成 Token
# GitHub → Settings → Developer settings → Personal access tokens
# 勾选：repo, public_repo

# 2. 配置 Git
git remote set-url origin https://username:TOKEN@github.com/username/repo.git

# 或使用 git credential helper
git config --global credential.helper store
```

**方案 B - 使用 SSH 密钥（推荐）：**
```bash
# 生成 SSH 密钥
ssh-keygen -t ed25519 -C "your@email.com"

# 复制到剪贴板
cat ~/.ssh/id_ed25519.pub | pbcopy

# 添加到 GitHub
# GitHub → Settings → SSH and GPG keys → New SSH key

# 配置 Git
git remote set-url origin git@github.com:username/repo.git
```

### 3. Python 环境冲突

**现象：**
```
ModuleNotFoundError: No module named 'xxx'
or
AttributeError: module 'xxx' has no attribute 'xxx'
```

**原因：**
- 多个 Python 版本冲突
- 虚拟环境未激活

**解决方案：**

**使用 virtualenv：**
```bash
# 创建虚拟环境
python3 -m venv venv

# 激活虚拟环境
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt

# 退出虚拟环境
deactivate
```

**方案 B - 使用 pyenv 管理多版本：**
```bash
# 安装 pyenv
brew install pyenv

# 安装 Python 版本
pyenv install 3.11.5

# 设置项目 Python 版本
echo "3.11.5" > .python-version

# 重新加载
pyenv rehash
```

### 4. clawhub 安装失败

**现象：**
```
Skill not found (remaining: 895/900, reset in 48s)
or
Error: Use --force to install suspicious skills
```

**原因：**
- 每日 API 调用次数限制
- 技能被标记为可疑

**解决方案：**

**方案 A - 等待刷新或批量安装：**
```bash
# 查看剩余调用次数
clawhub whoami

# 批量安装（避免单次调用过多）
clawhub install gog
clawhub install github
clawhub install summarize
```

**方案 B - 安装本地技能：**
```bash
# 克隆技能源码
git clone https://github.com/username/skill.git
cd skill

# 本地安装
cp -r . /path/to/openclaw/skills/skill
```

## 🔒 安全风险

### 1. 安装可疑技能

**现象：**
```
⚠️ Warning: "skill-name" is flagged as suspicious by VirusTotal
This skill may contain risky patterns (crypto keys, external APIs, eval, etc.)
```

**风险：**
- 🚨 **代码执行漏洞** - 恶意脚本
- 🚨 **数据泄露** - API 密钥窃取
- 🚨 **资源滥用** - 挖矿、DDoS

**防范措施：**

**方案 A - 检查 VirusTotal 评分：**
```bash
# 查看技能详情
clawhub inspect skill-name

# 检查源代码
cat skill/skill.py | grep -E "(eval|exec|__import__|subprocess)"
```

**方案 B - 沙箱测试：**
```bash
# 创建隔离环境
docker run -it --rm python:3.11 bash
# 在容器内测试技能
```

**方案 C - 最小权限原则：**
```bash
# 只授予必要权限
gog --scope read
# 不要授予：write, delete, admin
```

### 2. API 密钥泄露

**现象：**
```
git log | grep "API_KEY"
# 发现密钥被提交到仓库
```

**防范措施：**

**方案 A - 使用 .env 文件：**
```bash
# .env 示例
export GOOGLE_APPLICATION_CREDENTIALS="/path/to/key.json"
export GITHUB_TOKEN=ghp_your_token_here

# 加入 .gitignore
echo ".env" >> .gitignore
```

**方案 B - 使用环境变量管理：**
```bash
# ~/.bashrc
export OPENCLAW_API_KEY="your-key"

# 或使用 direnv
brew install direnv
echo 'load .env' > .envrc
direnv allow
```

### 3. 过度依赖云服务

**现象：**
- 所有数据上传到第三方
- 订阅费用失控
- 数据泄露风险

**防范措施：**

**方案 A - 本地优先：**
```bash
# 优先选择本地部署方案
brew install openclaw
clawhub install local-skill

# 云服务作为补充
clawhub install cloud-tool --optional
```

**方案 B - 定期审计：**
```bash
# 列出所有已安装技能
clawhub list

# 检查 API 调用频率
openclaw logs --api-calls
```

## ⚡ 性能优化

### 1. 技能过多导致响应慢

**现象：**
- AI 响应时间 > 10 秒
- CPU 使用率 > 80%

**解决方案：**

**方案 A - 精简技能：**
```bash
# 列出所有技能
clawhub list

# 卸载不常用技能
clawhub uninstall old-skill
```

**方案 B - 优先级排序：**
```python
# skill.py 中设置优先级
import argparse

parser.add_argument("--priority", type=int, default=0, help="技能优先级")
```

### 2. Git 仓库过大

**现象：**
```
fatal: /path/to/repo: cannot allocate memory
or
error: unable to read 100MB blob
```

**解决方案：**

**方案 A - 清理历史：**
```bash
# 使用 git filter-repo
pip install git-filter-repo
git filter-repo --strip-blobs-bigger-than 50M
```

**方案 B - 使用 Git LFS：**
```bash
git lfs install
git lfs track "*.zip"
git add .gitattributes
git add * .gitattributes
```
